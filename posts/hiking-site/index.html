做了一个广州徒步路线网站

2026年5月28日 3 分钟
徒步 广州 React Leaflet GitHub Pages

起因

手头有一份广州徒步路线的数据——越秀历史文化线和荔湾西关风情线，每条线路包含七八个途经点，每个点有坐标、介绍、照片点位。之前这些数据躺在一个 React 组件里，作为某个更大项目的一部分，但我觉得它们值得单独拿出来。

做成一个独立的徒步路线网站，链接到个人主页的徒步栏目下，方便自己和朋友用。

方案选型

目标很明确：一个纯粹的前端页面，带地图展示，零后端。部署在 GitHub Pages 上，不产生任何维护成本。

选了 Vite + React 做构建。原因很简单：数据本身就是 React 组件里的，移植成本最低。地图用了 Leaflet + OpenStreetMap，免费、轻量（40KB 左右），不需要注册 API key，对 GitHub Pages 这种纯静态部署非常友好。

AI 生成路线的功能保留了下来——用户可以输入偏好，让 AI 自动生成一条新线路。后端用一个 Cloudflare Worker 做 API 代理，保护 API key 不暴露在前端代码里。

结构

整个项目就一个页面，左右分栏布局：

左边是路线列表和行程详情。两条预设路线以卡片形式展示，点击切换。每个途经点可以展开查看介绍和核心亮点，红色编号和地图上的标记一一对应。

右边是地图，标注所有途经点，用路线色连线。点击标记弹出信息窗，展示景点名称和亮点。

配色上，每条路线有自己的主题色——越秀线用深红（#8B2020），呼应红砂岩城楼；荔湾线用藏蓝（#2A5C8A），接近西关大屋的青砖色调。主题色会贯穿整个页面：地图连线、标记编号、高亮边框、按钮，都跟着路线走。

底部有个 AI 定制按钮，点击后弹出输入框，填写偏好就能生成新路线。

部署

GitHub Pages 走得很顺。用 GitHub Actions 自动部署，推送 main 分支就触发构建发布。

部署后可以在 https://daitree42.github.io/hiking/ 访问。

Cloudflare Worker 部署了 DeepSeek API 的代理，把前端的 Anthropic 格式请求转为 DeepSeek 格式，响应再转回去。这样前端完全不需要修改。

几点感受

地图在徒步内容里是天然的一级界面。比起文字描述「从 A 步行 15 分钟到 B」，在地图上看到两个点的实际距离和相对位置要直观得多。后续可以考虑在路线数据里补充轨迹路径，让地图上显示的行走路线更精确。

两条预设路线覆盖了广州两千年的历史切片——从南越王的丝缕玉衣（公元前 2 世纪），到光孝寺的千年菩提树（东晋），到镇海楼的明代城墙（1380 年），再到中山纪念堂的民国风华（1931 年）。同一条路线上走完两千年，这种时空压缩感是广州这座城市特有的魅力。

AI 生成路线目前依赖 DeepSeek 的 API，生成质量取决于 prompt 的精细程度。后续可以沉淀一批 prompt 模板，或者把已知的景点数据做成知识库，让 AI 生成的路线更可靠。

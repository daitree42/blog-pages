Claude Code 练习项目推荐（初级）

2026年5月25日 4 分钟
Claude Code 练习项目

项目一：个人静态博客生成器

难度：★☆☆☆☆ | 技术栈：Python + Markdown + HTML

项目说明： 写一个命令行工具，把 Markdown 文件转换成静态 HTML 页面。支持首页文章列表、单篇文章页、简单的标签分类。

练习目标：

Claude Code 功能 在项目中的应用
Explore → Plan → Code → Commit 先分析需要什么功能，再出方案，再实现
CLAUDE.md 为项目写一个 CLAUDE.md，记录项目结构和约定
上下文管理 跨多个 session 开发，每次用 /compact 保持上下文干净
调试 故意留一个 bug，让 Claude 定位和修复

推荐理由： - 功能边界清晰，适合练习"给目标不给步骤"的 prompt 写法 - 涉及文件读写、目录遍历，Claude Code 的 agentic loop 优势明显 - 成品可用，有成就感

Prompt 示例：

在当前目录创建一个 Python 静态博客生成器。
输入是 _posts/ 目录下的 .md 文件（带 YAML frontmatter），
输出到 _site/ 目录的 HTML 文件。
首页显示文章列表，每篇文章有独立页面。
先用 Plan Mode 出方案。

项目二：命令行待办事项管理器（带数据持久化）

难度：★★☆☆☆ | 技术栈：任意语言（推荐 Python 或 Node.js）

项目说明： 一个在 terminal 中运行的待办事项工具，支持添加、完成、删除、列出任务，数据存储在 JSON 文件中。进阶可加搜索和优先级排序。

练习目标：

Claude Code 功能 在项目中的应用
迭代开发 分 2-3 轮逐步添加功能，练习 prompt 的增量式写法
Plan Mode 在加新功能前先用 Plan Mode 讨论方案
代码审查 让 Claude review 已写代码，检查边缘情况
测试 让 Claude 帮你写单元测试并运行

推荐理由： - CRUD 操作覆盖了大部分开发场景 - JSON 持久化涉及数据格式处理，Claude 的代码生成能力有发挥空间 - 功能可以逐步扩展，适合练习"分阶段"开发节奏

Prompt 示例：

用 Node.js 写一个 CLI 待办事项工具，支持：
- add <任务内容>
- done <id>
- list（按状态分组显示）
- delete <id>

数据存到 ~/.todos/todos.json。

先写核心功能，我确认后再加搜索功能。
每次改动后运行 npm test 确保不破坏已有功能。

项目三：个人主页 + 简历站点

难度：★★☆☆☆ | 技术栈：HTML + CSS + JavaScript（纯前端，可部署到 GitHub Pages）

项目说明： 制作一个个人展示页面，包含简介、技能列表、项目展示、联系方式。响应式设计，风格简洁。

练习目标：

Claude Code 功能 在项目中的应用
多文件项目管理 多个 HTML/CSS/JS 文件之间的协作
子代理 让一个子代理写页面结构，另一个写样式，并行工作
MCP 集成 练习连接外部资源（如 GitHub）
全流程闭环 从零开始到部署到 GitHub Pages 的完整流程

推荐理由： - 纯前端项目，不需要搭数据库或服务器，零障碍 - 可视化结果，正反馈强烈 - 可部署到 GitHub Pages，体验"从开发到上线"的完整闭环 - 子代理的并行任务在这里特别好用

Prompt 示例：

帮我从零创建一个个人主页站点。
布局：顶部导航 → Hero 区 → 技能卡片区 → 项目展示区 → 底部联系区。
配色简洁，响应式。

你负责搭结构和写样式，同时启动一个子代理去搜索现成的配色方案供参考。
完成后总结改动了哪些文件。

三个项目的对比

对比项 博客生成器 待办事项工具 个人主页
核心能力 工作流 + CLAUDE.md 迭代 + Plan Mode 多文件 + 子代理
代码量 ~200 行 ~150 行 ~300 行
完成时间 2-3 小时 1-2 小时 2-3 小时
是否可部署 是（生成静态文件） 否（CLI 工具） 是（GitHub Pages）
成就感 ★★★ ★★ ★★★★

学习路径建议

项目一：博客生成器
  → 重点学：Explore → Plan → Code → Commit 工作流
  → 重点学：CLAUDE.md 配置
      ↓
项目二：待办事项工具
  → 重点学：Plan Mode 做增量开发
  → 重点学：让 Claude 帮你写测试和 debug
      ↓
项目三：个人主页
  → 重点学：子代理并行工作
  → 重点学：全流程闭环（开发→部署）

按这个顺序做完三个项目，你会自然掌握 Claude Code 最核心的日常功能，之后再做实际项目就顺手了。

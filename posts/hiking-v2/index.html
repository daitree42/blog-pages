徒步网站改版记：加地图、修崩溃、适配国内网络

2026年5月29日 4 分钟
徒步 广州 React Leaflet GitHub Pages 踩坑

起因

上一次把广州徒步路线数据做成了独立站点，发布了。但没有地图的路线网站总觉得少了点什么——路线数据里明明有坐标，却没有用起来。

这次的目标很明确：把地图加上。结果打开了一个坑接一个坑的盖子。

加地图

用 Leaflet + react-leaflet，组件封装好，每条路线展开后在地图上显示途经点和连线。预设的两条路线（越秀历史文化线、荔湾西关风情线）各有七八个点，用编号标记一一标注，用路线主题色连线。

问题在于——数据里根本没坐标。

之前的数据只存了景点名、地址、介绍，lat 和 lng 字段是空的。Leaflet 拿到 undefined 坐标直接抛出 Invalid LatLng object: (undefined, undefined)，整个页面白屏。

给每条路线的每个站点加了真实经纬度。坐标来自公开的地理信息，13 个站点的位置都在广州老城区范围内。

白屏崩溃

加了坐标之后页面还是白屏。排查发现是因为 RouteMap 组件直接 import 了 Leaflet——而这个 import 发生在模块加载阶段，一旦 Leaflet 出问题（不管是版本兼容还是网络原因），整个 bundle 都执行不了，React 根本挂载不上。

解决方案是把 Leaflet 相关的代码拆成动态 import。RouteMap 只在卡片展开时才去加载地图 chunk，加载失败也不会影响主页面。同时加了 ErrorBoundary 组件兜底，捕获渲染异常并显示错误信息，而不是让用户面对一片空白。

旧路线数据兼容

「我的线路」功能保存用户之前 AI 生成的路线，但这些旧数据里没有坐标。切换到这个 tab 时，RouteMap 试图渲染 undefined 坐标，再次崩溃。

在 RouteMapImpl 里加了过滤逻辑：只渲染包含有效 lat/lng 的站点，过滤掉无效数据。同时修改了 AI 生成的系统提示词，要求返回坐标。

OSM 瓦片加载失败

地图渲染出来了，但显示的是灰色背景——图块没有加载。排查发现 OpenStreetMap 的瓦片服务器在国内网络环境下无法访问。

换成高德地图瓦片（webrd01.is.autonavi.com）。高德的瓦片在国内访问正常，而且对于国内城市的地图数据（道路标注、地点名称）反而比 OSM 更详细。改一行瓦片 URL 就解决了。

AI 生成降级

Cloudflare Worker 部署的 AI 代理不知什么原因停用了，前端请求一直 Failed to fetch。

临时方案是在前端加了一个本地备选生成器：预设了 12 个广州真实景点（光孝寺、陈家祠、沙面等），当远程 API 不可用时，随机组合这些景点生成路线。所有站点都带真实坐标，生成效果基本够用。

后来用户提供了 Cloudflare API 令牌和 DeepSeek API key，用 wrangler 重新部署了 Worker，AI 生成恢复正常。现在前端优先调用远程 API，失败则自动降级到本地生成。

链接到主页

最后把徒步网站的链接加到了个人主页的项目栏目下。主页也是 GitHub Pages 部署，改了 index.html 里徒步卡片的链接地址，自动生效。

改完后徒步网站可以稳定访问了：地图用高德瓦片，在国内正常显示；预设路线能展开看地图和步行路线；AI 生成有远程 API 和本地备选两层保障。链接挂在 https://daitree42.github.io/homepage/#projects 上。

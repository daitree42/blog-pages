Claude Code 学习经验总结

2026年5月25日
Claude Code AI

来源：Anthropic Skilljar — Claude Code 101 课程

一、核心理念：Claude Code 与普通聊天的区别

Claude Code 是一个 agentic loop（代理循环）系统，而不是简单的问答工具。它在 terminal 中运行，能够自主执行多步操作（读文件、搜索代码、运行命令），并且每一步都需要用户的 permission 控制。这意味着：

  • 它不是"你问它答"，而是"你给任务，它执行"
  • 它有自己的上下文窗口（context window）、工具集（tools）和权限系统（permissions）
  • 它可以看到你的文件系统，可以执行命令，可以编辑代码

二、核心工作流：Explore → Plan → Code → Commit

这是 Claude Code 最重要的方法论，任何功能开发都应该遵循这四步：

1. Explore（探索）

先让 Claude 阅读代码库，理解现有结构和模式，不要直接动手改。

2. Plan（规划）

在动手之前制定实现方案，规划涉及哪些文件、需要做什么改动。

3. Code（编码）

执行具体的编码工作。

4. Commit（提交）

完成后提交代码，附上清晰的提交信息。

关键经验：跳过前两步直接编码是新手最常见的错误。先理解再动手，效率反而更高。

三、上下文管理（Context Management）

Claude Code 的上下文窗口有限，随着对话增长需要主动管理：

命令 作用 使用场景
/compact 压缩历史，保留关键信息 对话变长、响应变慢时
/clear 清空整个对话历史 开始一个全新的任务时
/context 查看当前上下文中有什么 不确定 Claude 还记得多少时

关键经验：不要把 Claude 当持久化聊天伙伴。任务结束后该 compact 就 compact，该 clear 就 clear。Claude Code 的上下文由 CLAUDE.md 和 memory 系统来保持连续性，不靠聊天历史。

四、CLAUDE.md：项目的持久记忆

CLAUDE.md 是 Claude Code 最重要的自定义手段。它是一个位于项目根目录的 markdown 文件，每次 Claude 启动时自动读取。

适合放什么： - 项目结构和架构说明 - 编码规范和约定（命名规则、测试要求等） - 常用命令和工作流 - 项目背景和约束条件

使用技巧： - 创建 .claude/CLAUDE.md（项目级别，可提交到 git） - 也可以结合 memory 系统使用 - 内容越精确，Claude 的行为越符合预期

五、子代理（Subagents）与委派

当任务复杂时，不要在一个对话里做所有事。可以启动子代理来并行处理独立任务。

适用场景： - 需要同时搜索多个方向 - 有耗时较长的独立任务 - 不希望子任务的结果污染主窗口的上下文

关键经验：主上下文是宝贵资源。能用子代理做的事情就交给子代理。

六、MCP（Model Context Protocol）

MCP 允许 Claude Code 连接外部工具和数据源（如 API、数据库、文件系统等），是扩展 Claude 能力的主要方式。

七、Hooks：确定性控制

Hooks 是 shell 脚本，可以在特定事件触发时自动执行（如 tool call 之前/之后、提示提交时等）。

典型用途： - 自动格式化代码 - 拦截特定命令 - 发送通知 - 在读写文件时自动执行检查

关键经验：Hooks 提供的是确定性控制——不依赖模型判断，每次都执行。适合做格式检查、权限控制等需要绝对可靠的事情。

八、新人最需要注意的几点

  1. 权限模式：了解 approval mode 和 auto-accept 的区别，根据任务风险级别选择合适的模式
  2. Plan Mode 不是可选项：复杂任务先用 Plan Mode 让 Claude 输出方案，你再确认，再执行
  3. 不要把所有内容塞进提示词：把信息写在 CLAUDE.md 里，或者用 memory 系统，比写在提示词里更有效
  4. 隐私意识：上传到 Claude Code 的代码会被 Anthropic 处理，敏感信息要脱敏

九、推荐学习路径

  1. 先跑通安装，用 /init 初始化 CLAUDE.md
  2. 从一个简单的功能开始，完整走一遍 Explore → Plan → Code → Commit
  3. 练习用 /compact 和 /clear 管理上下文
  4. 为项目编写 CLAUDE.md，逐步完善
  5. 尝试用子代理做并行搜索
  6. 进阶：配置 hooks 实现自动化工作流
  7. 高阶：通过 MCP 集成外部工具

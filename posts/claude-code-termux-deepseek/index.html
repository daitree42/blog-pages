移动端極客實錄：在安卓 Termux 上完美運行 Claude Code（借殼 DeepSeek v4）

2026年5月26日 5 分钟
Claude Code Termux Android DeepSeek 教程

在大模型與 AI Agent 爆發的時代，Anthropic 官方推出的終端代碼助手 Claude Code 憑藉強大的多文件讀寫、自動 Git 管理和指令執行能力，成為了開發者的神器。然而，官方並沒有對安卓（Termux）環境提供原生支持，且其綁定的官方 API 價格不菲。

本文將完整記錄如何利用 PRoot 虛擬化技術 繞過架構限制，在手機上完美部署 Claude Code，並透過環境變量將其「靈魂」替換為極具性價比的 DeepSeek v4 API，打造一個隨身攜帶、價格低廉的全棧編程 Agent。

一、終端體驗優化：解決 Termux 中文輸入法 Bug

在 Termux 命令行與本地模型或 Agent 對話時，最常遇到的坑就是手機中文輸入法無法喚醒，或者打字時光標亂碼。這是因為移動端軟鍵盤在底層終端環境下默認鎖定了英文原始字符模式。

實戰最快的中文輸入方案

  • 使用 Termux 自帶的「文本輸入框」：在輔助按鍵欄（有 ESC、CTRL、ALT 的那一欄）向左滑動，找到 TEXT 按鈕。點擊它會彈出一個標準的 Android 文本輸入框。在這裡你可以像平時聊天一樣絲滑地打中文、換行、修改，點擊確定後，中文就會自動送入終端。
  • 修改強制限制作業（可選）：編輯 ~/.termux/termux.properties 配置文件，加入 enforce-char-based-input = true，隨後執行 termux-reload-settings 刷新，強制調用系統軟鍵盤。

二、核心搭建：在 PRoot Ubuntu 容器中部署 Claude Code

由於 Claude Code 包含大量需要編譯的原生硬體依賴，在原生安卓環境下會遇到權限隔離、/tmp 臨時路徑無法寫入等死結。最穩妥、後續升級最不易崩潰的方案是在 Termux 中部署一個虛擬的 Ubuntu 容器。

1. 安裝並穿越進入 Ubuntu 環境

打開 Termux，運行以下命令下載大管家並初始化系統：

pkg update && pkg install proot-distro
proot-distro install ubuntu
proot-distro login ubuntu

命令小科普：proot-distro login ubuntu 就像是一扇任意門。它利用 PRoot 技術，在不佔用額外內存、不讓手機發燙的前提下，把手機切換到了硬核的 Linux 電腦模式。

註：在安裝過程中如果彈出 Configuring tzdata 提示選擇時區，先敲數字 5 選擇 Asia（亞洲），再在城市列表裡找到 Shanghai（上海）或 Hong_Kong（香港）對應的序號回車即可。

2. 安裝開發基礎依賴

進入 Ubuntu 容器後（此時光標變為綠色的 root@localhost:~#），一鍵安裝 Node.js、npm 等必要工具：

apt update && apt install -y curl nodejs npm git ripgrep nano

3. 下載 Claude Code 並修復 Native Binary 報錯

執行官方推薦的獨立安裝腳本：

curl -fsSL https://claude.ai/install.sh | bash

此時如果直接輸入 claude，系統會因為官方沒有打包適配手機（arm64 架構）的原生二進制文件而甩出報錯：Error: claude native binary not installed.

核心修復命令：利用 npm 全局強制編譯，補回該架構的核心依賴：

npm install -g --unsafe-perm @anthropics/claude-code

4. 補齊快捷路徑

將編譯成功的運行路徑塞進系統搜索中：

echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc && source ~/.bashrc

三、借殼偷渡：配置 DeepSeek API Key

Claude Code 默認需要綁定官方付費帳戶，但幸運的是，DeepSeek 官方提供了完全兼容 Anthropic 協議格式的 API 接口。我們只需修改變量，就能完成這場低成本的「瞞天過海」。

修改內層 Ubuntu 的環境變量

在 Ubuntu 提示符下輸入：

nano ~/.bashrc

滑到文件最末尾，粘貼以下配置（記得將第一行的 Key 替換為你在 DeepSeek 後台申請的真實密鑰）：

# 填入你的真實 DeepSeek 秘钥
export ANTHROPIC_API_KEY="sk-xxxx你的真實deepseek密鑰xxxx"

# 修改 API 地址，指向 DeepSeek 的 Anthropic 兼容端點
export ANTHROPIC_BASE_URL="https://api.deepseek.com/anthropic"

# 指定要調用的模型（自動映射至 2026 最新推薦的 v4 全能模型）
export ANTHROPIC_MODEL="deepseek-v4-pro"

保存並退出（按 Ctrl + O → 回車 → Ctrl + X），然後刷新配置使其立即生效：

source ~/.bashrc

四、首次啟動與排雷指南

完成上述步驟後，直接在終端輸入命令喚醒系統：

claude

首次進入的正確選項（至關重要）

  1. 選擇皮膚主題（Theme）：默認勾選 2. Dark mode，直接按回車。
  2. 自定義 API Key 檢測：系統會提示 Detected a custom API key in your environment. Do you want to use this API key? - 注意：用鍵盤上方的方向鍵 ↑ 將光標切換到 1. Yes，然後按回車。允許它調用我們的 DeepSeek 憑證。
  3. 目錄安全審查：提示 Yes, I trust this folder，直接按回車信任當前的 /root 目錄。

五、日常使用的「正確姿勢」

在折騰過程中，我們曾嘗試使用快捷鍵（別名）想一步跨入系統，但因為「抄近路」會導致 Ubuntu 進入非登錄狀態、拒絕加載 ~/.bashrc 中的 DeepSeek 密鑰，從而引發連環報錯。

因此，日常在手機上使用 Claude Code，最穩妥、雷打不動的標準動作只有兩步：

第一步：打開 Termux，輸入命令正式登錄虛擬系統，讓 Ubuntu 把 DeepSeek 密鑰等所有配置整齊加載：

proot-distro login ubuntu

第二步：看到綠色的 root@localhost 提示符亮起後，直接喚醒代碼助手：

claude

進入對話界面後，你可以輸入 /status 驗證配置。當看到 Anthropic base URL 成功指向 api.deepseek.com 且 Model 顯示為 deepseek-v4-flash/pro 時，恭喜你，這套全網最極客、性價比拉滿的隨身 AI 編程工作站就徹底為你所用了！

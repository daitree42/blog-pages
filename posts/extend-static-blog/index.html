给纯 Python 静态博客加了四个功能

2026年5月27日 4 分钟
博客 静态站点 Python 搜索

上个月搭了一个零维护的静态博客，用纯 Python 把 Markdown 转成 HTML，推到 GitHub Pages 上。跑了几天，发现缺几个东西：标签只能看不能点、没有站点地图、想搜索得靠浏览器、改样式要等部署才能看效果。

花了一个晚上加了四个功能。核心思路不变——不加框架，不引入运行时依赖，能跑 Python 就行。

一、标签聚合页

之前 parse_draft() 已经把标签解析进了 article["tags"]，但 build() 里没有生成页面。加一段循环的事：

tag_map: dict[str, list[dict]] = {}
for a in articles:
    for tag in a.get("tags", []):
        tag_map.setdefault(tag, []).append(a)

tag_dir = DIST_DIR / "tag"
tag_dir.mkdir(parents=True)

for tag, tag_articles in tag_map.items():
    for p, total, page_items in paginate(tag_articles, PAGE_SIZE):
        render_paginated(
            "index.html", page_items, p, total,
            tag_dir / tag, f"tag/{tag}", ctx,
        )

复用首页模板，自动分页。现在文章标签可以点击了，比如 /tag/Python/ 列出所有带 Python 标签的文章。生成器会自动扫描所有标签，有文章就生成页面，没有就跳过。

二、Sitemap

Google Search Console 说没有 sitemap。确实没有。加一个：

sm_lines = [
    '<?xml version="1.0" encoding="UTF-8"?>',
    '<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">',
]

def sm_url(loc, lastmod="", priority="0.5"):
    block = f"  <url>\n    <loc>{SITE_URL}/{loc}</loc>\n"
    if lastmod:
        block += f"    <lastmod>{lastmod}</lastmod>\n"
    block += f"    <priority>{priority}</priority>\n  </url>"
    sm_lines.append(block)

sm_url("", priority="1.0")          # 首页
sm_url("archive/", priority="0.6")  # 存档
for cat_slug in CATEGORIES.values():
    sm_url(f"category/{cat_slug}/", priority="0.7")
for a in articles:
    sm_url(f"posts/{a['slug']}/", lastmod=a["date"].strftime("%Y-%m-%d"), priority="0.8")

sm_lines.append("</urlset>")
(DIST_DIR / "sitemap.xml").write_text("\n".join(sm_lines), encoding="utf-8")

优先级按层级分配：首页 1.0，栏目页 0.7，文章页 0.8。最后一篇新文章的 lastmod 日期也会自动填上。

三、全文搜索

本打算接一个第三方搜索服务，想了想不值得。静态博客的文章数量有限，构建时把所有文章的内容摘要导出成一个 JSON 文件，前端用 JavaScript 实时过滤就够了。

后端：构建时生成索引

index = []
for a in articles:
    plain = re.sub(r"<[^>]+>", "", a["body_html"])
    plain = html.unescape(plain).strip()
    excerpt = plain[:200].replace("\n", " ")
    index.append({
        "title": a["title"],
        "url": f"posts/{a['slug']}/",
        "date": a["date_display"],
        "tags": a["tags"],
        "category": a["category"],
        "excerpt": excerpt,
    })
(DIST_DIR / "search.json").write_text(
    json.dumps(index, ensure_ascii=False, indent=2), encoding="utf-8"
)

关键细节：用 re.sub 去掉 HTML 标签，取纯文本前 200 字做摘要。search.json 只有十几 KB，加载几乎无感。

前端：实时过滤

async function initSearch(inputId, resultsId) {
    var input = document.getElementById(inputId);
    var results = document.getElementById(resultsId);
    if (!input) return;

    var res = await fetch(PREFIX + "/search.json");
    var index = await res.json();

    input.addEventListener("input", function () {
        var q = input.value.trim().toLowerCase();
        results.innerHTML = "";
        if (!q) return;

        var hits = index.filter(function (a) {
            return a.title.toLowerCase().includes(q) ||
                a.excerpt.toLowerCase().includes(q) ||
                a.tags.some(function (t) { return t.toLowerCase().includes(q); });
        }).slice(0, 10);

        hits.forEach(function (a) {
            results.innerHTML += "" +
                '<a href="' + PREFIX + '/' + a.url + '" class="search-result">' +
                '<span class="search-title">' + a.title + "</span>" +
                '<span class="search-meta">' + a.date + " · " + a.category + "</span>" +
                '<span class="search-excerpt">' + a.excerpt + "</span></a>";
        });
    });
}

没有防抖（文章少不需要），没有高亮（保持简单），一次加载全部索引。打字即过滤，体验接近即时。

搜索框放在了导航栏下方，输入时下拉显示结果。样式只用了三四十行 CSS。

四、本地预览服务器

之前改一次样式就要 python3 publish.py && cd dist && python3 -m http.server，而且路径前缀 /blog-pages/ 还不匹配，得手动拼 URL。

加了一个 --serve 参数：

def serve(port: int = 8000):
    import http.server, socketserver

    class PrefixHandler(http.server.SimpleHTTPRequestHandler):
        def translate_path(self, path):
            if path.startswith(SITE_PREFIX + "/"):
                path = path[len(SITE_PREFIX):]
            return super().translate_path(path)

    with socketserver.TCPServer(("0.0.0.0", port), PrefixHandler) as httpd:
        httpd.serve_forever()

核心技巧是重写 translate_path，把 URL 里的 /blog-pages/ 前缀去掉，使本地路径与线上保持一致。现在一条命令即可：

python3 publish.py --serve --port 8080

打开 http://localhost:8080/blog-pages/，和线上看到的一模一样。

还顺手加了一个 --watch 模式，用 watchdog 库监听文件变动自动重建：

pip install watchdog
python3 publish.py --watch

尾记

这四个功能加起来不到 100 行新增代码。这是静态博客的好处：没有插件系统的学习成本，想要什么自己写。坏处也很明显：得会写。但对一个自己用的工具来说，这种掌控感很舒服。

整个 publish.py 现在不到 400 行，零外部运行时依赖（除了 markdown 和 jinja2 两个 apt 包）。理论上十年后回来，装两个包就能重新跑起来。
